# Security Policy

## Supported Versions

We actively support and fix security vulnerabilities in the following versions of the project:

| Version | Supported |
| ------- | --------- |
| 1.0.x   | Yes       |
| < 1.0   | No        |

---

## Reporting a Vulnerability

We take the security and integrity of our research tools seriously. If you find a security vulnerability, please do not disclose it publicly (such as by opening a public GitHub issue). Instead, follow the process below:

1. **Email the Maintainers**: Send a detailed security report to [security@pxalabs.org](mailto:security@pxalabs.org) (or contact the organization administrators).
2. **Include Details**: Please describe the vulnerability, include steps or code to reproduce it, and explain the potential impact.
3. **Response Time**: We will acknowledge receipt of your report within 48 hours and work to provide a resolution or patch as quickly as possible.

Thank you for helping keep this genomics research project secure!
